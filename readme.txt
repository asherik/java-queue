Реализована очередь на чистом java, без помощи rabbit и т.д.
Сервис разделен на 2 самостоятельные части - продюсер и консюмер.
Обе части работают по расписанию через каждые 2 секунды, их работа асинхронна.
В начале программы происходит добавление в очередь данных, ложится в глобальную переменную, которая также будет доступна для чтения второй частью сервиса.
В итоге видим при запуске программы, что задания ложатся продюсером в глобальную переменную (очередь). А консюмер сам вычитывает потом.
Взаимодействие выстроено по модели Point-to-Point и pub-sub
